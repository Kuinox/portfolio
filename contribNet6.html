Contributions .NET 6

En octobre 2020, j’ai remarqué un bug étrange sur un composant du framework .NET.
C’est peu étonnant étant donné que c’était sur une nouvelle fonctionnalité, et que contourner le bug pouvait se faire simplement.
On m’a confirmé que le comportement que j’observais était bien un bug sur une plateforme de discussions temps réel (Discord).
J’ai immédiatement ouvert un rapport de bug :
Voici le ticket pour ce bug: https://github.com/dotnet/runtime/issues/42999.
Je me propose aussi pour faire le correctif du bug. .NET étant open source, et acceptant les contributions externes, il est possible de proposer et écrire sois même les corrections de bugs et fonctionnalités.
Le ticket de bug m’a donc été assigné.
Après avoir expérimenté sur quelques weekend, j’ai finalement une correction possible.

Une revue de code est faite par un employé de Microsoft, après avoir fait les changements demandé la correction de bug a été ajouté dans la base de code. Cette correction de bug sera donc présente dans .NET 6.
Ce correctif permettait de retirer du code maintenant inutile, j’ai donc spontanément ouvert une 2ème contribution, qui est passée par le même processus et sera aussi présent en .NET 6.
