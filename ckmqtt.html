CK-MQTT

Mon premier projet autonome à Signature Code fut d’implémenter le protocole MQTT.
J’ai d’abord essayé d’adopter une base de code existante (MQTTnet), mais après plusieurs heures de travail dessus nous avons remarqué plusieurs erreurs qui mettent le doute sur le reste de la base de code.
J’ai donc abandonné cette base de code, et j’ai essayé d’en reprendre une autre, celle de Xamarin.
Xamarin est une entreprise connue dans le monde .NET et est connu pour avoir des excellents ingénieur.
Malheureusement, le code en question utilisait une style de programmation différent, et rendait difficile la maintenance, ajout de fonctionnalités, ou l’amélioration de performances, et montrait déjà des problèmes de performances.
J’ai donc décidé de partir de zéro, et d’implémenter tout moi-même.
Le protocole est censé être simple, et au premier abord, on pense naïvement que l’implémentation du protocole est lui aussi simple.
Mais à cause d’un manque de restrictions des spécifications, il est compliqué d’implémenter correctement le protocole, et à cause d’un manque de rigueur sur les implémentations existante, je n’ai vu aucune implémentation qui ne garantisse pas de ne pas causer de perte de donnée.
Aujourd’hui, le projet est en pause a un état avancé et manque encore du travail pour être robuste et stable.

Compétences associées :

  • Expertise C# et .NET
