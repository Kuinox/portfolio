CK-MQTT

Mon premier projet autonome à Signature Code fut d’implémenter le protocole MQTT.
J’ai d’abord essayé d’adopter une base de code existante (MQTTnet), mais après plusieurs heures de travail nous avons remarqué des erreurs qui mettaient le doute sur le reste de la base de code.
J’ai donc abandonné MQTTnet, et j’ai essayé d’en reprendre une autre, celle de Xamarin.
Xamarin est une entreprise connue dans le monde .NET et est connue pour avoir d'excellents ingénieurs.
Malheureusement, le code en question utilisait un style de programmation différent, et rendait difficile la maintenance, l'ajout de fonctionnalités, ou l’amélioration de performances, et montrait déjà des problèmes de performances.
J’ai donc décidé de partir de zéro, et d’implémenter tout moi-même.
Le protocole est censé être simple, et au premier abord, on pense naïvement que l’implémentation du protocole est lui aussi simple.
Mais à cause d’un manque de precisions des spécifications, il est compliqué d’implémenter correctement le protocole.
De plus à cause d’un manque de rigueur sur les implémentations existantes, je n’ai vu aucune implémentation garantissant l'absence de perte de donnée.
Aujourd’hui, le projet est à un état bien avancé mais en standby.

Compétences associées :

  • Expertise C# et .NET
