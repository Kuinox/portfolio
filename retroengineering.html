Retro ingénierie

Avoir des connaissance de base en la rétroingénierie est très utile pour un développeur. A plusieurs occasion, cela m’a été utile dans un cadre professionnel.
Lorsque Fortnite a commencé a devenir populaire, j’y ai beaucoup joué avec mes amis.
Etant extrêmement intéressé dans les données générée au cours d'une partie, je voulais voir s’ils y avaient des donnés exploitable dans les fichier de replay.
Le code source du moteur de jeu de Fortnite est disponible à la lecture, mais il est licensé.
Pour connaitre le contenu du replay légalement, il faut donc uniquement inspecter le contenu du fichier, on appelle ça de la rétro ingénierie de boite noire.
Cela prend énormément de temps, et m’ayant désintéressé de Fortnite, je n’étais plus interessé de poursuivre dessus, mais cela à énormément aider la communauté qui s’était formé dans le but de lire les replays.
https://github.com/Kuinox/ChartsNite

Wireshark

Wireshark permet d’inspecter ce qu’une machine émet, et reçois. Ce logiciel m’a été particulièrement utile pour débugger des protocoles de communications que j’implémentais, mais m’a permis aussi de trouver beaucoup plus tôt que le problème n’étais pas de mon coté mais avec la machine tiers avec laquelle je communiquais.

Hex Editeur

Un editeur hex, pour hexadecimal, permet d’afficher le contenu brut d’un fichier, même si ce n’est pas du texte.
Il est utile pour savoir qu’est ce qui est enregistrer dans un fichier, comparer, et voir des détails pas forcément visible normalement. Il permet par exemple de voir les charactères de texte invisible, etc.
