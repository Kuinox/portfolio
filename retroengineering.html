Rétro ingénierie

Avoir des connaissances de base en la rétro-ingénierie est un atout pour un développeur.
À plusieurs occasions, cela m’a été utile dans un cadre professionnel.
Lorsque Fortnite a commencé à devenir populaire, j’y ai beaucoup joué avec mes amis.
Étant extrêmement intéressé dans les données générées au cours d'une partie, je voulais voir si les fichiers de replay contenaient des informations exploitables.
Le code source du moteur de jeu de Fortnite est disponible à la lecture, mais il a une licence restrictive.
Je ne pouvais donc pas légalement lire le code source pour m'aider.
Pour faire une rétro-ingénierie légale, je ne pouvais deviner le contenu du fichier qu'en inspectant le contenu.
On appelle ça de la rétro-ingénierie de boite noire.
La difficulté réside dans le fait que le format de la donnée enregistrée est inconnu, c'est comme si je devais déchiffrer une langue inconnue.
J'ai fait des progrès conséquents sur la compréhension du format, mais cela nécessitait énormément de temps, et m’étant désintéressé de Fortnite, je ne souhaitait plus poursuivre mes recherches sur le sujet.
Mes efforts ont néanmoins permis à la communauté de pouvoir extraire l'entièreté des informations disponibles dans ces fichiers.
Lien vers une librairie permettant de lire les replay fortnite.

Wireshark

Wireshark permet d’inspecter ce qu’une machine émet, et reçoit.
Ce logiciel m’a été particulièrement utile pour débugger des protocoles de communication que j’implémentai.
Cela permet d'immédiatement identifier de quel côté réside le problème: parfois, le problème n'est pas dans notre implémentation, mais vient de la machine avec laquelle on communique.

Hex Editeur

Un éditeur hex, pour hexadecimal, permet d’afficher et éditer le contenu brut d’un fichier, même si ce n’est pas du texte.
Il est utile pour inspecter ce qui est enregistré dans un fichier, et voir des détails normalement invisibles. Il permet par exemple de voir les caractères de texte invisible, etc.

Visual Studio

Visual Studio est un excellent outil de développement, mais ses outils de debugging peuvent être utilisés pour comprendre le fonctionnement de programmes.
